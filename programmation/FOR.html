Qu'est-ce qu'une Boucle for ?

La boucle for ("Pour") est une structure de contrôle qui permet de répéter un bloc de code un nombre de fois précis.

Elle est idéale lorsque vous savez à l'avance combien de fois vous voulez exécuter une action (par exemple, "faire clignoter une LED 5 fois").

La syntaxe décortiquée

La boucle for est composée de trois parties entre parenthèses, séparées par des points-virgules :

for (initialisation; condition; incrémentation) {
  // ... code à répéter ...
}
  • Initialisation : Exécutée une seule fois au tout début. On y crée généralement une variable qui servira de compteur (ex: int i = 0).
  • Condition : Évaluée avant chaque tour de boucle. Si elle est vraie, le tour de boucle s'exécute. Si elle est fausse, la boucle s'arrête.
  • Incrémentation : Exécutée à la fin de chaque tour de boucle. Elle sert le plus souvent à mettre à jour le compteur (ex: i++ pour ajouter 1 à i).

Exemple 1 : Compter de 0 à 3

// Fait 4 tours de boucle (pour i = 0, 1, 2, 3)
for (int i = 0; i < 4; i++) {
  Serial.print("Le compteur i vaut : ");
  Serial.println(i);
}

Exemple 2 : Faire un décompte

On peut aussi décrémenter (i--) au lieu d'incrémenter.

// Compte à rebours de 3 à 1
for (int i = 3; i > 0; i--) {
  Serial.println(i);
  delay(1000); // Attend 1 seconde
}
Serial.println("Décollage !");

Piège 1 : Le point-virgule parasite

Comme pour le if, un point-virgule (;) après la parenthèse termine l'instruction. La boucle va "tourner dans le vide" et le bloc d'accolades qui suit ne sera exécuté qu'une seule fois, après la fin de la boucle.

// ERREUR : Le ; termine le for
for (int i = 0; i < 5; i++); 
{
  // Ce bloc n'est plus dans la boucle !
  // Il s'exécutera une seule fois, quand i vaudra 5.
  Serial.println(i); // Affichera "5" une seule fois.
}

Piège 2 : L'erreur "off-by-one" (décalage d'un)

C'est l'erreur la plus commune avec les boucles. Attention à votre condition d'arrêt !

  • i < 5 : Boucle pour i = 0, 1, 2, 3, 4. (5 itérations)
  • i <= 5 : Boucle pour i = 0, 1, 2, 3, 4, 5. (6 itérations)
// On veut 10 répétitions.
// Ce code en fait 11 ! (de 0 à 10 inclus)
for (int i = 0; i <= 10; i++) {
  // ...
}
// Le bon code est : for (int i = 0; i < 10; i++)

Piège 3 : Modifier le compteur dans la boucle

Il est fortement déconseillé de modifier la variable du compteur (ici, i) à l'intérieur du bloc d'accolades. Cela peut entraîner des boucles infinies ou un comportement totalement imprévisible.

// BOUCLE INFINIE !
for (int i = 0; i < 10; i++) {
  Serial.println(i);
  i = 0; // i est remis à 0 à chaque tour, il n'atteindra jamais 10 !
}
