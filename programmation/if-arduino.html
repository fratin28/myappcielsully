Comprendre l'Instruction if Arduino

Prendre des décisions dans votre code.

Qu'est-ce qu'une Instruction if ?

L'instruction if ("Si") est la structure de contrôle la plus importante. Elle permet à votre programme de prendre une décision.

Elle exécute un bloc de code uniquement si une condition est vraie. Si la condition est fausse, le bloc est ignoré.

Syntaxe 1 : if (simple)

Le cas le plus basique : "Si cela est vrai, fais ceci."

int temperature = 25;

if (temperature > 20) {
  // Ce code s'exécute, car 25 > 20 est VRAI
  Serial.println("Il fait chaud !");
}

Syntaxe 2 : if...else (Si... Sinon)

Permet de choisir entre deux blocs : "Si cela est vrai, fais ceci. SINON, fais cela."

int age = 15;

if (age >= 18) {
  // Ce bloc est ignoré (15 >= 18 est FAUX)
  Serial.println("Vous êtes majeur.");
} else {
  // Ce bloc s'exécute
  Serial.println("Vous êtes mineur.");
}

Syntaxe 3 : if...else if...else (La chaîne)

Permet de tester plusieurs conditions à la suite. Le programme s'arrête au premier bloc vrai qu'il trouve.

int note = 12;

if (note < 10) {
  // Faux (12 < 10 est FAUX)
  Serial.println("Insuffisant");
} else if (note < 14) {
  // Vrai (12 < 14 est VRAI)
  // Ce bloc s'exécute
  Serial.println("Mention passable");
} else if (note < 18) {
  // Ignoré (même si 12 < 18 est vrai, on s'est arrêté avant)
  Serial.println("Mention bien");
} else {
  // Ignoré
  Serial.println("Mention très bien");
}

Piège 1 : = (Assignation) vs == (Comparaison)

C'est l'erreur la plus difficile à trouver !

  • x == 5 : "Est-ce que x est ÉGAL à 5 ?" (Condition, ce qu'on veut)
  • x = 5 : "Mets 5 DANS x" (Assignation, change la valeur)
int x = 10;
// ERREUR ! Ceci ne teste pas, il ASSIGNE 5 à x.
// L'assignation (5) est VRAIE (non-zéro).
if (x = 5) { 
  // Ce bloc s'exécute toujours, et x vaut 5 après !
  Serial.println("Erreur !");
}
// Le bon code est : if (x == 5) { ... }

Piège 2 : Le point-virgule parasite

Un point-virgule (;) termine une instruction. Si vous en mettez un après le if, vous créez un bloc... vide !

// ERREUR : Le ; termine le if
if (temperature < 10); 
{
  // Ce bloc n'est plus lié au 'if'.
  // Il s'exécutera TOUJOURS !
  Serial.println("Toujours affiché");
}

Piège 3 : Oublier les accolades { }

Si vous n'utilisez pas d'accolades, le if ne contrôle que la toute première ligne qui le suit.

int x = 10;
if (x > 5)
  Serial.println("Ligne 1"); // Contrôlée par le if
  Serial.println("Ligne 2"); // PAS contrôlée ! S'exécute toujours.

// Si x valait 0, le code afficherait "Ligne 2",
// ce qui est une erreur de logique.
// Mettez toujours des accolades !
