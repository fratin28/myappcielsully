Qu'est-ce qu'une Boucle while ?

La boucle while ("Tant que") répète un bloc de code tant qu'une condition spécifiée est vraie. Elle est parfaite quand on ne sait pas à l'avance combien de tours de boucle seront nécessaires, par exemple pour attendre une action de l'utilisateur.

Le fonctionnement est simple : 1. On teste la condition. 2. Si elle est vraie, on exécute le code. 3. On retourne à l'étape 1. Si la condition est fausse dès le départ, la boucle ne s'exécute jamais.

Syntaxe de while

// On initialise une variable avant la boucle
int compteur = 0;

// On teste la condition ici
while (compteur < 5) { // Tant que compteur est inférieur à 5...
  Serial.println(compteur);
  
  // TRÈS IMPORTANT : il faut modifier la variable de la condition !
  compteur++; 
}
// Le code arrive ici quand compteur vaut 5.

Testez la logique while !

Simulons un capteur qui doit atteindre une valeur seuil. La boucle tournera "tant que" la valeur lue est inférieure au seuil.

int valeurLue = 0;
int seuil = [Votre Valeur];
while (valeurLue < seuil) {
  // ... on incrémente la valeur pour la simulation
  valeurLue += 10;
}

Piège 1 : La Boucle Infinie

C'est l'erreur la plus courante avec while. Si la condition de la boucle ne devient jamais fausse, votre programme restera bloqué à l'intérieur pour toujours !

int i = 0;
// ERREUR ! i n'est jamais modifié dans la boucle.
// La condition (i < 10) sera donc toujours vraie.
while (i < 10) {
  Serial.println("Au secours, je suis bloqué !");
  // On a oublié d'écrire "i++;"
}

Piège 2 : Quand utiliser while ou for ?

Les deux peuvent parfois accomplir la même chose, mais il y a une règle simple :

  • Utilisez for quand vous savez combien de fois vous devez répéter (ex: "clignoter 5 fois").
  • Utilisez while quand vous répétez jusqu'à ce qu'un événement se produise (ex: "attendre qu'un bouton soit pressé").
